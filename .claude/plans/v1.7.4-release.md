## T1: Instance-isolate context_usage.json

**Problem:** Statusline writes `~/.empirica/context_usage.json` as a singleton.
Multiple Claude instances overwrite each other's context percentage.

**Files to change:**
1. `~/.claude/plugins/local/empirica/scripts/statusline_empirica.py` (writer, ~line 946)
2. `~/.claude/plugins/local/empirica/hooks/context-shift-tracker.py` (reader, ~line 80)

**Implementation:**
- Import `_get_instance_suffix()` from `project_resolver` in both files
- Writer: `context_usage{suffix}.json` instead of `context_usage.json`
- Reader: same suffix pattern to read the right file
- Both already have access to the lib path pattern

**Verification:**
- Run statusline in two tmux panes, confirm separate files created
- Check context-shift-tracker reads correct instance file

**Commit:** `fix(hooks): Instance-isolate context_usage.json for multi-pane support`

## T2: Read dedup in Sentinel

**Problem:** AI re-reads the same files multiple times in a transaction, wasting context window.
Sentinel already tracks `edited_files` in hook_counters — extend to `read_files`.

**Files to change:**
1. `~/.claude/plugins/local/empirica/hooks/sentinel-gate.py` — add read tracking

**Implementation:**
- On PostToolUse for Read tool: append file_path to `read_files` list in hook_counters
- On PreToolUse for Read tool: check if file_path already in `read_files`
- If duplicate: allow but add advisory note in `permissionDecisionReason`
  - "Re-reading {path} (read {N}x this transaction). Consider using cached knowledge."
- Do NOT block — purely informational, same as autonomy nudges
- Track count per path (dict not list) to show frequency

**Verification:**
- Read the same file twice in a transaction, confirm nudge appears on second read
- Confirm no blocking behavior

**Commit:** `feat(sentinel): Advisory read-dedup tracking in hook_counters`

## T3: Plugin version sync on pip upgrade

**Problem:** `pip install -e .` updates the empirica package but the plugin at
`~/.claude/plugins/local/empirica/` is a separate directory tree. Hook code
lives ONLY in the plugin (18 files), NOT in the source repo (0 files).

**Design decision needed:** The plugin IS the source of truth for hooks.
The "version sync" issue is really about:
1. Plugin `plugin.json` version field not matching `pyproject.toml`
2. No mechanism to detect version drift between CLI and plugin

**Implementation options (pick one):**
- **Option A:** Post-install hook in pyproject.toml that updates plugin.json version
- **Option B:** CLI `empirica doctor` command that checks plugin version alignment
- **Option C:** Session-init hook checks CLI version vs plugin.json, warns on mismatch

**Recommendation:** Option C — lightweight, no build system changes, immediate feedback.

**Files to change:**
1. `~/.claude/plugins/local/empirica/hooks/session-init.py` — add version check
2. `~/.claude/plugins/local/empirica/plugin.json` — ensure version field exists

**Verification:**
- Temporarily set plugin.json version to "0.0.0", start session, confirm warning
- Set back to correct version, confirm no warning

**Commit:** `feat(hooks): CLI/plugin version drift detection at session start`

## T4: Post-compact token budget

**Problem:** Recovery context after compaction consumes ~50K tokens (5% of 1M).
Breakdown: summary ~10K, system prompt ~6K, MEMORY.md ~4K, re-injected skill ~8K,
re-injected file contents ~7K, boilerplate ~10K.

**What we control:**
- `post-compact.py` — our injected recovery prompt (3 prompt builders, all max_items=5)
- `pre-compact.py` — what we capture before compaction
- MEMORY.md — size of the index file

**What we DON'T control (Claude Code behavior):**
- Conversation summary verbosity
- Re-injection of file contents read before compaction
- Skill re-injection
- System-reminder boilerplate

**Implementation:**
1. **Audit current injection size** — measure token count of post-compact output
2. **Add token ceiling to prompt builders** — truncate at N tokens (target: 2K max)
3. **MEMORY.md pruning** — add line count check, warn if >100 lines
4. **Pre-compact: flag files as ephemeral** — investigate if Claude Code honors
   any signal to NOT re-inject file contents (likely not, but worth checking)
5. **Document findings** — what's in our control vs Claude Code's

**Verification:**
- Compact and measure context% immediately after recovery
- Target: <3% (30K tokens) total recovery overhead

**Commit:** `perf(hooks): Cap post-compact recovery injection with token budget`
